# User Path Report

Streamlit-приложение для отчета по путям пользователей на основе Yandex Metrika Logs API. MVP v0.1 отвечает на один вопрос: куда пользователи идут после выбранной страницы и какие пути приводят к выбранной цели.

## Что делает приложение

- Загружает несемплированные данные Logs API по обязательному URL-фильтру.
- Находит визиты с выбранной страницей.
- Загружает hits только для найденных `visitID`, чтобы собрать полный путь внутри визита.
- Показывает сводку, следующий шаг после выбранной страницы, топ путей и список `visitID` для ручной проверки.

## Установка и запуск

```bash
pip install -r requirements.txt
streamlit run app.py
```

## Токен Яндекс Метрики

Токен не хранится в коде. Добавьте его в Streamlit Secrets:

```toml
YANDEX_METRIKA_TOKEN = "..."
```

Также можно задать переменную окружения `YANDEX_METRIKA_TOKEN`.

## Почему нужен URL-фильтр

Приложение не загружает весь счетчик без фильтра. Это защищает от тяжелых запросов, ускоряет отчет и снижает риск превысить ограничения Logs API. URL-фильтр обязателен для первого шага загрузки visits.

## Почему лучше выбирать один завершенный день

По умолчанию выбран вчерашний день. Текущий день не используется, потому что данные могут быть неполными, а незавершенные визиты и задержки обработки могут искажать пути и конверсию.

## Источник данных

Отчет строится по Yandex Metrika Logs API и не использует семплированные интерфейсные отчеты Метрики.

## Ограничения v0.1

- Нет Sankey-диаграмм.
- Нет AI-выводов.
- Отчет показывает только следующие шаги и топ путей.
- Если hits по `visitID` не загрузились, полный путь построить нельзя.
- Нет Вебвизора, score, priority_score и сложных фильтров.
