# User Path Report

Streamlit-приложение для отчета по путям пользователей на основе Yandex Metrika Logs API. MVP v0.1 отвечает на один вопрос: куда пользователи идут после выбранной страницы и какие пути приводят к выбранной цели.

## Что делает приложение

- Сначала создает маленькую выгрузку `hits` по обязательному URL-фильтру и минимальным полям `visitID`, `dateTime`, `URL`.
- Находит уникальные `visitID`, где был выбранный URL, и показывает их количество.
- Затем загружает `visits` и full `hits` только по найденным `visitID` через фильтр `IN (...)` с лимитом 100 / 500 / 1000.
- Показывает сводку, следующий шаг после выбранной страницы, топ путей и список `visitID` для ручной проверки.

## Установка и запуск

```bash
pip install -r requirements.txt
streamlit run app.py
```

## Токен Яндекс Метрики

Токен не хранится в коде. Добавьте его в Streamlit Secrets:

```toml
YANDEX_METRIKA_TOKEN = "..."
```

Также можно задать переменную окружения `YANDEX_METRIKA_TOKEN`.

## Почему отчет начинается с hits

Приложение не начинает отчет с выгрузки `visits` по URL: для больших счетчиков такой запрос может оказаться слишком широким, а клиентская фильтрация после скачивания не помогает, если Logs API не дает создать log request. Первый шаг всегда ищет выбранный URL в `hits` на сервере, получает только `visitID`, `dateTime`, `URL`, а затем ограниченно загружает `visits` и full `hits` по найденным `visitID`. Если Logs API отвечает `requested log is too big`, приложение останавливается и просит ввести полный URL или сузить период, не делая fallback на выгрузку всего счетчика.

В сайдбаре есть кнопка «Очистить подготовленные log requests». Если очистка через API недоступна или возвращает ошибку, очистите подготовленные log requests в Метрике/API или подождите, пока они будут удалены.

## Почему лучше выбирать один завершенный день

По умолчанию выбран вчерашний день. Текущий день не используется, потому что данные могут быть неполными, а незавершенные визиты и задержки обработки могут искажать пути и конверсию.

## Источник данных

Отчет строится по Yandex Metrika Logs API и не использует семплированные интерфейсные отчеты Метрики.

## Ограничения v0.1

- Нет Sankey-диаграмм.
- Нет AI-выводов.
- Отчет показывает только следующие шаги и топ путей.
- Если hits по `visitID` не загрузились, полный путь построить нельзя.
- Нет Вебвизора, score, priority_score и сложных фильтров.
